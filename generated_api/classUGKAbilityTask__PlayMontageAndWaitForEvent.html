class UGKAbilityTask_PlayMontageAndWaitForEvent : public UAbilityTask

This task combines PlayMontageAndWait and WaitForEvent into one task, so you can wait for multiple types of activations such as from a melee combo Much of this code is copied from one of those two ability tasks This is a good task to look at as an example when creating game-specific tasks It is expected that each game will have a set of game-specific tasks to do what they want

static UGKAbilityTask_PlayMontageAndWaitForEvent *PlayMontageAndWaitForEvent(UGameplayAbility *OwningAbility, FName TaskInstanceName, UAnimMontage *MontageToPlay, FGameplayTagContainer EventTags, const FGameplayAbilityTargetDataHandle &Data, float Rate = 1.f, FName StartSection = NAME_None, bool bStopWhenAbilityEnds = true, float AnimRootMotionTranslationScale = 1.f)

Play a montage and wait for it end. If a gameplay event happens that matches EventTags (or EventTags is empty), the EventReceived delegate will fire with a tag and event data. If StopWhenAbilityEnds is true, this montage will be aborted if the ability ends normally. It is always stopped when the ability is explicitly cancelled. On normal execution, OnBlendOut is called when the montage is blending out, and OnCompleted when it is completely done playing OnInterrupted is called if another montage overwrites this, and OnCancelled is called if the ability or task is cancelled

Parameters

  • TaskInstanceName – Set to override the name of this task, for later querying

  • MontageToPlay – The montage to play on the character

  • EventTags – Any gameplay events matching this tag will activate the EventReceived callback. If empty, all events will trigger callback

  • Rate – Change to play the montage faster or slower

  • bStopWhenAbilityEnds – If true, this montage will be aborted if the ability ends normally. It is always stopped when the ability is explicitly cancelled

  • AnimRootMotionTranslationScale – Change to modify size of root motion or set to 0 to block it entirely
