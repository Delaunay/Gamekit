24 /* Queued Ability
25  *
26  * For an Ability to be queuable, all the data it requires
27  * needs to be available when the Ability is first queued AND
28  * at the time of execution
29  *
30  * So moving to a point on the map is fine.
31  * Casting a spell on an ennemy will only be possible if
32  * the enemy is in range & visible at the time.
33  *
34  * The queued ability saves all the data it requires
35  * and will skip some activation steps (like requesting targets)
36  *
37  * Instead the data will be validated before execution
38  */
