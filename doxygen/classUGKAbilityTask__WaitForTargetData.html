FGameplayTag  ActivationTag

private

Valid TargetData was replicated to use (we are server, was sent from client)

Call into the TargetActor to sanitize/verify the data. If this returns false, we are rejecting the replicated target data and will treat this as a cancel.

This can also be used for bandwidth optimizations. OnReplicatedTargetDataReceived could do an actual trace/check/whatever server side and use that data. So rather than having the client send that data explicitly, the client is basically just sending a 'confirm' and the server is now going to do the work in OnReplicatedTargetDataReceived.
